# Feature Specification: iPadゲーム体験改善 — サウンド・ステージ拡張・演出強化

**Feature Branch**: `002-ipad-game-enhancements`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "iPadのゲーム体験を改善する。音が出ていないので、プログラム生成の効果音とBGMを実装（Web Audio APIのOscillatorNodeで生成。音声ファイル不要）、ステージが短すぎるので長くする、ステージ数を3→8に増やし太陽系の星々を巡る旅にする（月→火星→木星→土星→天王星→海王星→冥王星→太陽）、星を取った時にパーティクルバーストの演出を追加、現在のステージ名をHUD上部に表示する"

## Assumptions

- 対象デバイスはiPad専用（横向き固定）。既存の001-space-travel-gameがベース
- サウンドはすべてプログラム生成で実装し、外部音声ファイルは使用しない
- BGMはステージごとに異なるテーマ／トーンを持ち、子供向けのポップで明るい雰囲気にする
- ステージの長さは現在の値（500〜900）から体感で2〜3倍程度に延長し、遊びごたえを増す
- ステージ順は太陽系の旅をテーマとする：月→火星→木星→土星→天王星→海王星→冥王星→太陽
- 最終ステージ（太陽）は特別感のあるクライマックスとして設計する
- パーティクルバーストは星の色に合わせた演出で、虹色の星はより派手な演出にする
- 既存のゲームオーバーなし（隕石に当たってもスピードが落ちるだけ）の設計は変更しない
- 難易度はステージが進むにつれ徐々に上がる（隕石の出現頻度増加、星の密度変化）

## User Scenarios & Testing *(mandatory)*

### User Story 1 — ステージプレイ中にBGMが流れる (Priority: P1)

ステージが始まると自動的にBGMが流れ始め、宇宙の旅の雰囲気を盛り上げる。BGMはプログラムで生成されたシンセサウンドで、子供が楽しめるポップで明るいメロディ。ステージクリア時やタイトル画面に戻ったときにBGMは適切に切り替わる。

**Why this priority**: 音がまったく出ていない状態はゲーム体験として最も大きな欠落であり、BGMはゲーム全体の雰囲気を形成する最重要要素。

**Independent Test**: ステージを開始し、BGMが自動的に再生されること、ステージクリア時にBGMが停止または切り替わることを確認する。

**Acceptance Scenarios**:

1. **Given** タイトル画面が表示されている, **When** 画面が表示されている間, **Then** タイトル用のBGMが流れている
2. **Given** ステージが開始された, **When** 宇宙船が前進を始める, **Then** そのステージに対応したBGMが自動的に流れ始める
3. **Given** BGMが再生中, **When** ステージをクリアする, **Then** BGMが停止しクリア演出のジングルに切り替わる
4. **Given** iPad Safariでゲームを起動する, **When** 最初のユーザータッチ操作後, **Then** AudioContextが正しくresumeされBGMが再生される

---

### User Story 2 — 星を取った時に効果音が鳴る (Priority: P1)

宇宙船が星に触れて収集した瞬間、心地よい効果音が鳴る。通常の星と虹色の星で異なる効果音が再生される。隕石に衝突したとき、ブーストを発動したとき、ステージクリア時にもそれぞれ効果音が鳴る。

**Why this priority**: 効果音は操作へのフィードバックとして不可欠で、特に星の収集音はゲームの達成感に直結する。

**Independent Test**: ステージをプレイし、星に接触した瞬間に効果音が鳴ること、隕石衝突・ブースト・クリア時にもそれぞれ異なる効果音が鳴ることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** 宇宙船が通常の星に触れる, **Then** 明るいキラキラした収集音が鳴る
2. **Given** ステージをプレイ中, **When** 宇宙船が虹色の星に触れる, **Then** 通常の星とは異なるより華やかな収集音が鳴る
3. **Given** ステージをプレイ中, **When** 宇宙船が隕石に衝突する, **Then** 衝突音が鳴る
4. **Given** ステージをプレイ中, **When** ブーストボタンを押す, **Then** 加速感のあるブースト音が鳴る
5. **Given** ステージの終端に到達した, **When** ステージをクリアする, **Then** 達成感のあるクリアジングルが鳴る

---

### User Story 3 — 8つのステージで太陽系を巡る (Priority: P1)

ゲーム全体を通して太陽系の8つの天体を巡る壮大な旅を体験できる。月から始まり、火星、木星、土星、天王星、海王星、冥王星を経て、最終目的地の太陽を目指す。各ステージはその天体の特徴をイメージした雰囲気を持ち、ステージが進むにつれ適度に難しくなる。

**Why this priority**: ステージ数の拡張はゲームのボリュームと満足感に直結し、太陽系というテーマは子供の知的好奇心も刺激する。

**Independent Test**: ステージ1（月）からステージ8（太陽）まで順番にプレイし、各ステージの目的地名・難易度・ステージの長さが適切に設定されていることを確認する。

**Acceptance Scenarios**:

1. **Given** ゲームを開始した, **When** ステージ1をプレイする, **Then** 目的地は「月」で、最も低い難易度のステージが始まる
2. **Given** ステージ1をクリアした, **When** 次のステージに進む, **Then** ステージ2の目的地は「火星」で、ステージ1よりやや難しくなっている
3. **Given** 各ステージをクリアしていく, **When** ステージが進む, **Then** 月→火星→木星→土星→天王星→海王星→冥王星→太陽の順で各天体を巡る
4. **Given** ステージ7（冥王星）をクリアした, **When** ステージ8に進む, **Then** 最終ステージ「太陽」が始まり、特別な緊張感のあるステージになっている
5. **Given** ステージ8（太陽）をクリアした, **When** 全ステージをクリアする, **Then** エンディング画面が表示され太陽系の旅を完遂した達成感がある

---

### User Story 4 — ステージが十分な長さで遊びごたえがある (Priority: P2)

各ステージが以前より長くなり、宇宙の旅をじっくり楽しめる。星を集めたり隕石を避けたりする場面が十分にあり、短すぎてすぐ終わる感覚がない。それでも子供が飽きない適切な長さに調整されている。

**Why this priority**: ステージの長さは満足感の基盤だが、BGMやステージ数があってこそ効果を発揮するためP2とする。

**Independent Test**: 各ステージをプレイし、開始から目的地到着までに十分な時間と操作機会があることを体感で確認する。

**Acceptance Scenarios**:

1. **Given** ステージ1を開始する, **When** プレイを続ける, **Then** 以前の約2〜3倍の長さがあり、星や隕石と十分にインタラクションできる
2. **Given** 後半のステージをプレイする, **When** ステージ8までプレイする, **Then** 後半のステージほど長くなり、クリアまでの達成感が増している
3. **Given** 全ステージをプレイする, **When** 各ステージの所要時間を比較する, **Then** ステージの進行に合わせて適切に長さが増加している

---

### User Story 5 — 星を取った時にパーティクルバーストが出る (Priority: P2)

宇宙船が星を収集した瞬間、星の位置からカラフルなパーティクルが弾けるように飛び散る視覚演出が表示される。通常の星は金色のパーティクル、虹色の星はより大きくカラフルなパーティクルバーストになる。パーティクルは短時間で消滅し、ゲームプレイを邪魔しない。

**Why this priority**: 視覚的フィードバックはゲームの気持ちよさに貢献するが、ゲームの進行やコンテンツ量に影響しないためP2とする。

**Independent Test**: ステージをプレイし、通常の星と虹色の星をそれぞれ収集して、パーティクルバーストのサイズ・色・消滅タイミングが適切であることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** 宇宙船が通常の星に触れる, **Then** 星の位置から金色のパーティクルが放射状に飛び散る
2. **Given** ステージをプレイ中, **When** 宇宙船が虹色の星に触れる, **Then** 通常の星より大きく、多色のパーティクルが派手に飛び散る
3. **Given** パーティクルバーストが発生した, **When** 短時間（約0.5〜1秒）が経過する, **Then** パーティクルが完全に消滅し、画面上に残骸が残らない
4. **Given** 複数の星を連続で取得した, **When** パーティクルが同時に複数発生する, **Then** ゲームのパフォーマンスに影響なく滑らかに表示される

---

### User Story 6 — HUD上部に現在のステージ名が表示される (Priority: P2)

ステージプレイ中、画面上部のHUD領域に現在の目的地（ステージ名）が表示される。例えば「🌙 月をめざせ！」のように天体の絵文字と目的地名がひらがなで表示され、子供が今どこに向かっているのかひと目でわかる。

**Why this priority**: ステージ名の表示は8つに増えたステージの区別に役立つが、ゲームプレイ自体に影響しないためP2とする。

**Independent Test**: 各ステージを開始し、HUD上部に正しいステージ名（天体名）が表示されていることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージ1が開始された, **When** HUDが表示される, **Then** HUD上部に「🌙 月をめざせ！」のようなステージ名が表示される
2. **Given** 各ステージをプレイする, **When** ステージごとにHUDを確認する, **Then** そのステージの目的地に対応した天体名と絵文字が正しく表示されている
3. **Given** HUD上部にステージ名が表示されている, **When** プレイ中ずっと, **Then** ステージ名はスコアや星の数の表示と干渉せず読みやすい位置にある

---

### Edge Cases

- iPad Safariの自動再生制限により、ユーザーの最初のタッチ操作までAudioContextがsuspended状態の場合がある。最初のタッチでresumeする
- AudioContextの初期化に失敗した場合、音なしでもゲームプレイは正常に継続する（音なしフォールバック）
- パーティクルが大量に同時発生した場合（連続で星を取得）、パフォーマンスが劣化しないよう最大同時パーティクル数に上限を設ける
- 8ステージすべてクリア済みの状態でゲームを再開した場合、ステージ1からやり直せる
- ステージの長さが延びることで、セーブ／再開機能が中断位置ではなくステージ単位で動作する点は変更しない

## Requirements *(mandatory)*

### Functional Requirements

**サウンド — BGM**

- **FR-001**: システムはプログラム生成によるBGMを再生しなければならない（外部音声ファイル不要）
- **FR-002**: タイトル画面、各ステージ、エンディング画面にそれぞれ対応したBGMが流れなければならない
- **FR-003**: シーン遷移時にBGMが適切に停止・切り替えされなければならない
- **FR-004**: iPad Safariの自動再生制限に対処し、ユーザーの最初のタッチ操作後に音声の再生を開始しなければならない
- **FR-005**: 音声の初期化に失敗した場合でも、ゲームプレイは正常に継続しなければならない（音なしフォールバック）

**サウンド — 効果音（SFX）**

- **FR-006**: 星を収集した時、効果音が再生されなければならない（通常の星と虹色の星で異なる音）
- **FR-007**: 隕石に衝突した時、衝突効果音が再生されなければならない
- **FR-008**: ブーストを発動した時、ブースト効果音が再生されなければならない
- **FR-009**: ステージクリア時、クリアジングルが再生されなければならない
- **FR-010**: すべての効果音はプログラム生成されなければならない（外部音声ファイル不要）

**ステージ構成**

- **FR-011**: ゲームは8つのステージで構成されなければならない：月→火星→木星→土星→天王星→海王星→冥王星→太陽の順
- **FR-012**: 各ステージの長さは従来の約2〜3倍に延長されなければならない
- **FR-013**: ステージが進むにつれ、隕石の出現頻度が増加し、難易度が上がらなければならない
- **FR-014**: 最終ステージ（太陽）は他のステージとは異なる特別感のある設定でなければならない
- **FR-015**: 全8ステージをクリアするとエンディング画面が表示されなければならない

**パーティクルバースト**

- **FR-016**: 星を収集した瞬間、星の位置からパーティクルバーストが発生しなければならない
- **FR-017**: 通常の星は金色のパーティクル、虹色の星は多色のパーティクルバーストでなければならない
- **FR-018**: パーティクルは短時間（約0.5〜1秒）で消滅しなければならない
- **FR-019**: 同時に複数のパーティクルバーストが発生してもパフォーマンスに影響がないようにしなければならない

**HUDステージ名表示**

- **FR-020**: ステージプレイ中、HUD上部に現在の目的地名が表示されなければならない
- **FR-021**: ステージ名は対応する天体の絵文字とひらがなのテキストで表示されなければならない
- **FR-022**: ステージ名の表示は既存のスコア・星の数の表示と干渉してはならない

### Key Entities

- **Stage（ステージ）**: 太陽系の各天体に対応する8つのゲームステージ。番号・目的地名・ステージの長さ・難易度パラメータを持つ
- **SynthSound（合成サウンド）**: プログラム生成される音。BGMとSFXの2種類があり、それぞれ周波数・波形・音量変化で特徴づけられる
- **ParticleBurst（パーティクルバースト）**: 星の収集時に発生する視覚演出。位置・色・パーティクル数・寿命を持つ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: すべてのステージでBGMが途切れなく再生される
- **SC-002**: 星の収集・隕石衝突・ブースト・ステージクリアの全4種類のイベントで効果音が再生される
- **SC-003**: 8つのステージすべてが正しい順序（月→火星→木星→土星→天王星→海王星→冥王星→太陽）でプレイできる
- **SC-004**: 各ステージのプレイ時間が従来の約2〜3倍になっている
- **SC-005**: 星を取得するたびにパーティクルバーストが表示され、1秒以内に消滅する
- **SC-006**: 全ステージのHUD上部に正しいステージ名（天体名＋絵文字）が表示される
- **SC-007**: iPad Safari上で音声の自動再生制限を適切に処理し、ユーザー操作後に音が鳴る
- **SC-008**: パーティクルバーストが連続発生してもiPad上で滑らかに（目視で引っかかりなく）動作する

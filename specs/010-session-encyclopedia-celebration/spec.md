# Feature Specification: セッション管理・ずかん永続化・全クリアお祝い演出

**Feature Branch**: `010-session-encyclopedia-celebration`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "Safari スワイプ終了リセット、ずかん永続化、全クリアお祝い演出、ブースト炎確認"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Safari スワイプ終了時にゲームを最初から開始 (Priority: P1)

子どもが Safari をスワイプして閉じた後、再び Safari を開いてゲームにアクセスしたとき、ゲームは最初のステージ（ステージ1）から始まる。現状では localStorage に保存されたセーブデータ（clearedStage）が Safari 再起動後も残るため、途中のステージから再開してしまう。セッション管理によりゲームの進行データをリセットし、毎回新鮮な体験を提供する。

**Why this priority**: ユーザーの「スワイプしたらリセットして最初から遊びたい」という基本的な操作期待に直結する。これが実現しないとゲームの再プレイ体験が損なわれ、他の機能改善の効果も半減する。

**Independent Test**: Safari を開いてゲームを数ステージ進める → Safari をスワイプ終了 → Safari を再起動してゲームにアクセス → ステージ1から開始されることを確認。

**Acceptance Scenarios**:

1. **Given** プレイヤーがステージ5までクリアした状態で Safari をスワイプ終了した, **When** Safari を再起動してゲームを開く, **Then** ゲームはステージ1から開始される
2. **Given** プレイヤーがゲームをプレイ中である, **When** Safari のタブを閉じずにホーム画面に戻り、再度 Safari に戻る, **Then** ゲームの進行状態は維持されている（スワイプ終了していないため）
3. **Given** プレイヤーが初めてゲームを開いた, **When** ゲームが読み込まれる, **Then** セッションが新規作成され、ステージ1から開始される
4. **Given** プレイヤーがステージ3をクリアした直後, **When** 次のステージに進む, **Then** 進行データ（clearedStage = 3）がセッション内で正しく反映され、ステージ4から開始される

---

### User Story 2 - ずかん（図鑑）データの永続化 (Priority: P2)

プレイヤーが全ステージクリア後も、解放済みの惑星ずかん（unlockedPlanets）が保持される。現状では EndingScene で saveManager.clear() を呼び出しているため、全クリア時にずかんデータも消えてしまう。全クリア時にはゲーム進行（clearedStage）のみリセットし、ずかんは永続的に保持する。ただし Safari をスワイプ終了した場合は、ずかんも含めてすべてリセットされる（US1 と連動）。

**Why this priority**: ずかんは子どもの達成感・コレクション欲求に直結する。全クリア後にずかんが消えるのはユーザー体験上の重大な問題であり、US1（セッション管理）の次に対処すべき。US1 のセッション管理基盤の上に構築される。

**Independent Test**: ゲームを全クリア → エンディング表示 → タイトルに戻る → ずかんを開く → 全惑星がアンロック済みであることを確認。その後 Safari をスワイプ終了 → 再起動 → ずかんが空であることを確認。

**Acceptance Scenarios**:

1. **Given** プレイヤーが全11ステージをクリアしてエンディングに到達した, **When** エンディングが表示される, **Then** ゲーム進行（clearedStage）は0にリセットされるが、ずかんデータ（unlockedPlanets）は全て保持される
2. **Given** 全クリア後にタイトル画面に戻った, **When** ずかんボタンを押す, **Then** これまでに解放された全ての惑星が表示される
3. **Given** 全クリア後にゲームを再度プレイし始めた, **When** ステージ1をクリアする, **Then** ステージ1の惑星が既にずかんに存在しても重複登録されない
4. **Given** ずかんに全11惑星が登録済みである, **When** Safari をスワイプ終了して再起動する, **Then** ずかんデータも含めて全てリセットされ、ずかんは空になる（US1 と連動）
5. **Given** 5ステージ分の惑星がずかんに登録済みで、Safari をスワイプ終了せずにタブを維持している, **When** ゲームに戻る, **Then** ずかんの5惑星は保持されている

### User Story 3 - 全クリア時のエイリアンお祝い演出 (Priority: P3)

全11ステージをクリアしてエンディングに到達すると、これまでに出会った全11体のエイリアンコンパニオンが画面に登場し、お祝いの演出を行う。エイリアンが円形に並ぶ → 各エイリアンが順番にポップイン → 全員でジャンプ（バウンス）→「みんな ありがとう！」テキスト表示、という一連の演出で子どもの達成感を最大化する。

**Why this priority**: 全クリアの達成感を盛り上げる演出であり、ゲーム体験の締めくくりとして重要。ただし US1・US2 のデータ管理が整っていることが前提であり、機能的な優先度はそれらより低い。

**Independent Test**: ゲームを全クリアしてエンディングに到達 → エイリアンが順番に登場し、お祝い演出が最後まで再生されることを目視確認。

**Acceptance Scenarios**:

1. **Given** プレイヤーが全11ステージをクリアした, **When** エンディング画面が表示される, **Then** 全11体のエイリアンコンパニオンが画面上に登場する
2. **Given** エンディング画面が表示された, **When** 演出が開始される, **Then** エイリアンたちが円形のフォーメーションに並ぶ
3. **Given** エイリアンが円形に並んでいる, **When** 登場アニメーションが再生される, **Then** 各エイリアンが順番に拡大しながら「パッ」と現れる（1体ずつ時間差で登場）
4. **Given** 全エイリアンが登場完了した, **When** 祝福アニメーションが開始される, **Then** 全エイリアンが一斉にジャンプ（上下バウンス）する動きを繰り返す
5. **Given** 祝福アニメーションが表示されている, **When** 演出が進行する, **Then** 「みんな ありがとう！」テキストが画面に表示される
6. **Given** お祝い演出が全て完了した, **When** プレイヤーが画面をタップする, **Then** タイトル画面に戻る

---

### User Story 4 - ブースト炎の持続動作確認 (Priority: P4)

Feature 009 で実装したブースト炎の持続表示とフェードアウト演出が正しく動作していることを確認する。コード上は StageScene.ts の update() 内で `if (this.boostSystem.isActive()) { this.emitFlameParticles(); }` と実装されており、emitFlameParticles() 内にフェードアウトロジック（残り0.5秒で放出数・サイズ減少）も含まれている。本 User Story は新規実装ではなく、既存実装が仕様通りに動作していることをテストで保証する。

**Why this priority**: 既にコード上は正しく実装済みであり、新規開発は不要。テストによる動作保証の確認のみであり最低優先度。

**Independent Test**: 既存のブースト炎テストスイートを実行し、全テストがパスすることを確認。

**Acceptance Scenarios**:

1. **Given** ブースト炎の実装が Feature 009 で完了している, **When** ブーストを発動する, **Then** 炎エフェクトがブースト有効期間中途切れなく表示される（既存テストでカバー済み）
2. **Given** ブーストの残り時間が0.5秒以下になった, **When** 炎エフェクトを観察する, **Then** 放出数とサイズが段階的に減少してフェードアウトする（既存テストでカバー済み）
3. **Given** Feature 009 のテストスイートが存在する, **When** テストスイートを実行する, **Then** ブースト炎関連のテストが全てパスする

---

### Edge Cases

- Safari をスワイプ終了ではなく、タブを閉じただけの場合は sessionStorage が残るため進行データは維持される
- localStorage への書き込みが失敗した場合（ストレージ上限・プライベートブラウジング等）、ずかんデータの保存に失敗してもゲーム進行自体はブロックしない
- エンディング演出中に Safari がバックグラウンドに移行した場合、フォアグラウンド復帰後もアニメーションが再開または完了状態で表示される
- 全クリア前に一部のステージだけクリアした状態で Safari をスワイプ終了した場合、ずかんも進行データも両方リセットされる
- sessionStorage が利用不可な環境（極端に古いブラウザ等）では、フォールバックとして従来通り localStorage のみで動作する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ゲーム起動時に sessionStorage にセッションフラグが存在しない場合、ゲーム進行データ（clearedStage）とずかんデータ（unlockedPlanets）を全てリセットしなければならない
- **FR-002**: ゲーム起動時に sessionStorage にセッションフラグを設定しなければならない
- **FR-003**: セッション内でのゲーム進行データ（clearedStage）およびずかんデータ（unlockedPlanets）は localStorage に保存し続けなければならない
- **FR-004**: 全クリア時（エンディング到達時）にゲーム進行データ（clearedStage）を0にリセットしなければならない
- **FR-005**: 全クリア時（エンディング到達時）にずかんデータ（unlockedPlanets）は削除せず保持しなければならない
- **FR-006**: エンディング画面で全11体のエイリアンコンパニオンが画面上に表示されなければならない
- **FR-007**: エイリアンは円形フォーメーションに配置され、各エイリアンが順番に拡大して登場するアニメーションを再生しなければならない
- **FR-008**: 全エイリアン登場後、一斉にジャンプ（上下バウンス）する祝福アニメーションを再生しなければならない
- **FR-009**: 祝福アニメーション中に「みんな ありがとう！」テキストを画面に表示しなければならない
- **FR-010**: Feature 009 で実装済みのブースト炎の持続表示・フェードアウト動作がテストで保証されていなければならない

### Key Entities

- **セッションフラグ**: sessionStorage に保持するフラグ。Safari スワイプ終了で自動クリアされる。存在有無でセッション継続/新規を判定する
- **ゲーム進行データ（clearedStage）**: クリア済みステージ番号。localStorage に保存。セッション新規時・全クリア時にリセット
- **ずかんデータ（unlockedPlanets）**: 解放済み惑星番号の配列。localStorage に保存。セッション新規時にはリセット、全クリア時には保持
- **エイリアンコンパニオン**: 全11体の惑星エイリアン。エンディング演出で円形配置・ポップイン・バウンスアニメーションを行う

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Safari スワイプ終了後の再起動で、100%の確率でゲームがステージ1から開始される
- **SC-002**: Safari のタブ維持状態（スワイプ終了なし）では、ゲーム進行が100%保持される
- **SC-003**: 全クリア後にずかんを確認すると、全11惑星が表示される（欠落0件）
- **SC-004**: 全クリア後に再プレイを開始すると、ステージ1から開始される
- **SC-005**: エンディング演出で全11体のエイリアンが画面に登場し、お祝いアニメーションが完了まで再生される
- **SC-006**: エンディング演出の全工程（登場 → ポップイン → バウンス → テキスト表示）が途切れなく再生される
- **SC-007**: ブースト炎関連の既存テストスイートが全てパスする
- **SC-008**: iPad Safari 上で全機能が正常に動作する

## Assumptions

- Safari スワイプ終了により sessionStorage が自動的にクリアされる（iPad Safari の標準動作に依拠）
- sessionStorage は iPad Safari のプライベートブラウズモードでも利用可能である（制限がある場合はフォールバック動作で対応）
- ゲームの全ステージ数は11で固定（PLANET_ENCYCLOPEDIA に11エントリ）
- CompanionManager.createCompanionMesh() が全11惑星に対し正常にメッシュを生成できる
- エンディング演出は既存の starfield 背景の手前に表示する
- ブースト炎の持続・フェードアウトは Feature 009 の実装がそのまま有効であり、コード変更は不要

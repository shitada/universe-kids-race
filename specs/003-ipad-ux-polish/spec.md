# Feature Specification: iPadゲーム体験改善 第2弾 — UXポリッシュ

**Feature Branch**: `003-ipad-ux-polish`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "iPadのゲーム体験を改善する第2弾。サウンド再生修正、HUDステージ名表示修正、ホームボタン追加、チュートリアル画面追加、ブーストクールダウンインジケーター追加、ブーストボタンデザイン改善"

## Assumptions

- 対象デバイスはiPad（横向き固定）。既存の002-ipad-game-enhancementsのコードがベース
- iPad SafariのAudioContext自動再生制限が根本原因。`AudioContext.resume()`をユーザージェスチャーの**同期コールバック内**で呼び出す必要がある（現在はasync/awaitで非同期化されているため、ジェスチャーとの紐付けが切れている可能性が高い）
- HUDのステージ名表示は、現在のDOM構造とCSSで描画はされているが、z-indexやpointer-events、または表示位置の問題でiPad実機上では視認できていない可能性がある
- ホームボタンはHUDの一部として配置し、ステージプレイ中のみ表示する。タップでタイトル画面に遷移する
- チュートリアル画面は独立したオーバーレイとして実装する。短いアニメーションで操作方法を説明し、閉じるとタイトル画面に戻る
- ブーストのクールダウンは現在5秒間。この時間の進捗をインジケーターで可視化する
- ブーストボタンのデザイン変更は見た目のみの変更であり、機能的な挙動は変更しない
- 子供向けのゲームであるため、すべてのUIはひらがな表記で明るく楽しいデザインにする

## User Scenarios & Testing *(mandatory)*

### User Story 1 — iPad Safariでサウンドが確実に再生される (Priority: P1)

iPad Safariでゲームを起動し「あそぶ」ボタンをタップした後、BGMと効果音がすべて正常に再生される。現在はAudioContext.resume()がユーザージェスチャーの同期コールバック外で実行されているため、iPad Safariで音が鳴らない。ユーザーの最初のタッチ操作の同期コールバック内でAudioContextのresumeを行い、それ以降のすべての音声再生を確実に動作させる。

**Why this priority**: 音が鳴らない問題はゲーム体験の根幹を損なうバグであり、最優先で修正が必要。

**Independent Test**: iPad Safariでゲームを起動し「あそぶ」をタップ後、BGMが流れ始めること、星の収集音・隕石衝突音・ブースト音・クリアジングルがすべて鳴ることを確認する。

**Acceptance Scenarios**:

1. **Given** iPad Safariでゲームを開いた状態, **When** 「あそぶ」ボタンをタップする, **Then** AudioContextがresumedになり、BGMが再生される
2. **Given** AudioContextがresumed状態, **When** ステージをプレイする, **Then** 星収集・隕石衝突・ブースト・クリアの全SFXが正常に再生される
3. **Given** AudioContextの初期化に失敗した場合, **When** ゲームを続行する, **Then** 音なしでもゲームプレイは正常に継続する

---

### User Story 2 — HUD上部にステージ名が正しく表示される (Priority: P1)

ステージプレイ中、画面上部のHUD領域に「🌙 つきを めざせ！」のようなステージ名が確実に表示される。iPad実機で視認可能な位置・サイズ・z-indexで描画され、他のHUD要素（スコア・星の数）と重ならない。

**Why this priority**: プレイヤーが現在どのステージにいるか分からないのはUX上の重大な欠陥であり、すでに実装済みの機能が表示されていない不具合修正。

**Independent Test**: iPad実機で各ステージを開始し、HUD上部にステージ名が表示され、ゲームプレイ終了まで視認可能であることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージが開始された, **When** HUDが表示される, **Then** HUD上部に天体の絵文字とステージ名がひらがなで表示される
2. **Given** HUDにステージ名が表示されている, **When** ステージをプレイ中, **Then** ステージ名はスコアや星の数と重ならず読みやすい
3. **Given** iPad Safariで表示している, **When** ステージ名の要素を確認する, **Then** 適切なz-index・フォントサイズ・色で表示されiPad画面上で視認できる

---

### User Story 3 — ゲーム中にタイトル画面に戻れるホームボタン (Priority: P1)

ステージプレイ中、HUD上にホームボタン（家のアイコン等）が表示されている。このボタンをタップすると、現在のステージを中断してタイトル画面に戻る。子供が遊び終わりたい時やステージを変えたい時に、いつでもタイトルに戻れる安心感を提供する。

**Why this priority**: ステージ中にタイトルに戻る手段がないのは致命的なナビゲーション欠陥であり、ゲームの基本操作性に関わる。

**Independent Test**: ステージプレイ中にホームボタンをタップし、タイトル画面に遷移することを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** HUDを確認する, **Then** ホームボタンが視認しやすい位置に表示されている
2. **Given** ステージをプレイ中, **When** ホームボタンをタップする, **Then** ステージが中断されタイトル画面に遷移する
3. **Given** ホームボタンをタップした, **When** タイトル画面に戻った後, **Then** 進行状況（クリア済みステージ）は保持されている
4. **Given** ゲームプレイ中, **When** 誤ってホームボタンに触れないよう, **Then** ボタンはプレイ操作（左右移動・ブースト）と干渉しない位置に配置されている

---

### User Story 4 — タイトル画面から操作方法を確認できるチュートリアル (Priority: P2)

タイトル画面の右下に「あそびかた」ボタンがある。タップすると操作方法を説明するチュートリアル画面が開く。左右タッチで宇宙船の移動、ブーストボタンの使い方、星を集めてゴールを目指すことを、短いシンプルなアニメーションで視覚的に説明する。チュートリアルは短く（数秒程度のループまたは自動進行）、閉じるボタンでタイトル画面に戻れる。

**Why this priority**: 初めて遊ぶ子供が操作方法を理解できないと遊べないが、直感的な操作のため重要度はP2。

**Independent Test**: タイトル画面で「あそびかた」をタップし、チュートリアル画面が表示されアニメーションで操作説明が行われること、閉じてタイトル画面に戻れることを確認する。

**Acceptance Scenarios**:

1. **Given** タイトル画面が表示されている, **When** 画面を確認する, **Then** 右下に「あそびかた」ボタンが表示されている
2. **Given** タイトル画面が表示されている, **When** 「あそびかた」ボタンをタップする, **Then** チュートリアル画面が表示される
3. **Given** チュートリアル画面が表示されている, **When** アニメーションを見る, **Then** 左右タッチでの移動方法が視覚的に示される
4. **Given** チュートリアル画面が表示されている, **When** アニメーションを見る, **Then** ブーストボタンの使い方が視覚的に示される
5. **Given** チュートリアル画面が表示されている, **When** アニメーションを見る, **Then** 星を集めてゴールを目指すことが視覚的に示される
6. **Given** チュートリアル画面が表示されている, **When** 閉じるボタンをタップする, **Then** タイトル画面に戻る

---

### User Story 5 — ブーストのクールダウン進捗がインジケーターで分かる (Priority: P2)

ブーストを使用した後、ブーストボタンの下にクールダウンの進捗を示すインジケーター（円形またはバー）が表示される。クールダウン中はインジケーターが徐々に満たされていき、完了した瞬間に光るエフェクトで「使えるようになった」ことが分かる。子供がブーストの再使用タイミングを直感的に把握できる。

**Why this priority**: ブーストが使えるかどうかのフィードバックはゲーム体験を向上させるが、なくてもプレイ可能であるためP2。

**Independent Test**: ステージでブーストを使用し、クールダウン中にインジケーターが進捗表示されること、クールダウン完了時に光るエフェクトが発生することを確認する。

**Acceptance Scenarios**:

1. **Given** ブーストを使用した, **When** クールダウンが開始される, **Then** ブーストボタンの下にクールダウン進捗インジケーターが表示される
2. **Given** クールダウン中, **When** 時間が経過する, **Then** インジケーターが徐々に満たされ、残り時間が視覚的に分かる
3. **Given** クールダウンが完了した, **When** ブーストが再使用可能になる, **Then** 光るエフェクトが発生しブーストが使えることが視覚的に伝わる
4. **Given** ブーストがまだ使用されていない（初期状態）, **When** ステージ開始直後, **Then** インジケーターは「使用可能」状態で表示される

---

### User Story 6 — ブーストボタンが可愛くて派手なデザインになる (Priority: P3)

ブーストボタンが現在のシンプルなボーダーデザインから、子供が押したくなるような可愛くて派手なデザインに変わる。絵文字（🚀やキラキラ等）、グラデーション背景、押した時のアニメーション（拡大・光る等）で、ゲームの楽しさを視覚的に演出する。

**Why this priority**: 見た目の改善は楽しさに貢献するが、機能的な影響はないためP3。

**Independent Test**: ステージプレイ中にブーストボタンの外観を確認し、グラデーション・絵文字・押下時アニメーションが適用されていることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** ブーストボタンを確認する, **Then** 絵文字を含む可愛いデザインで表示されている
2. **Given** ステージをプレイ中, **When** ブーストボタンを確認する, **Then** グラデーション背景が適用されている
3. **Given** ブーストが使用可能な状態, **When** ブーストボタンをタップする, **Then** 押下時に拡大や光るなどのアニメーションが再生される
4. **Given** ブーストがクールダウン中, **When** ブーストボタンを確認する, **Then** 使用不可であることが視覚的に分かる（半透明化やグレーアウト等）

---

### Edge Cases

- AudioContextがブラウザのセキュリティポリシーにより完全にブロックされる環境では、音なしでゲームを正常に継続する
- ホームボタンとブーストボタンが隣接しすぎて誤タップしないよう、十分な間隔を確保する
- チュートリアル画面の閉じるボタンが小さすぎて子供がタップしにくくならないよう、適切なサイズにする
- ブーストのクールダウンインジケーターがゲームのパフォーマンスに影響を与えないこと（毎フレームのDOM更新を最小限に）
- 画面回転や画面サイズ変更時にHUD要素（ホームボタン、インジケーター等）の位置が崩れないこと
- チュートリアル画面を開いた状態でブラウザをバックグラウンドにし、復帰した時にアニメーションが正常に動作すること

## Requirements *(mandatory)*

### Functional Requirements

**サウンド再生修正（iPad Safari対応）**

- **FR-001**: `AudioContext.resume()`をユーザージェスチャーの同期コールバック内で実行しなければならない（async/awaitによる非同期化を避ける）
- **FR-002**: AudioContextの初期化とresumeが完了した後にBGM再生を開始しなければならない
- **FR-003**: AudioContext初期化失敗時でもゲームプレイは正常に継続しなければならない

**HUDステージ名表示修正**

- **FR-004**: ステージ名表示要素がiPad Safari上で確実に視認可能でなければならない（z-index、position、font-size、colorの適切な設定）
- **FR-005**: ステージ名はHUD上部の中央に配置され、スコア・星の数と重ならないように表示しなければならない

**ホームボタン**

- **FR-006**: ステージプレイ中、HUDにホームボタンを表示しなければならない
- **FR-007**: ホームボタンのタップでタイトル画面に遷移しなければならない
- **FR-008**: ホームボタンはゲーム操作（左右移動・ブースト）と干渉しない位置に配置しなければならない
- **FR-009**: ホームボタンによるタイトル遷移時、クリア済みステージの進行状況は保持しなければならない

**チュートリアル画面**

- **FR-010**: タイトル画面の右下に「あそびかた」ボタンを表示しなければならない
- **FR-011**: チュートリアル画面で左右タッチによる移動操作をアニメーションで説明しなければならない
- **FR-012**: チュートリアル画面でブーストボタンの使い方をアニメーションで説明しなければならない
- **FR-013**: チュートリアル画面で星を集めてゴールを目指す目的をアニメーションで説明しなければならない
- **FR-014**: チュートリアル画面に閉じるボタンがあり、タップでタイトル画面に戻れなければならない
- **FR-015**: チュートリアルのアニメーションはシンプルかつ短い（数秒程度）ものでなければならない

**ブーストクールダウンインジケーター**

- **FR-016**: ブースト使用後、ブーストボタンの下にクールダウン進捗インジケーターを表示しなければならない
- **FR-017**: インジケーターはクールダウン時間（5秒）の経過に応じて進捗を表示しなければならない
- **FR-018**: クールダウン完了時に光るエフェクトを表示しなければならない
- **FR-019**: インジケーターの更新がゲームのパフォーマンスに悪影響を与えてはならない

**ブーストボタンデザイン改善**

- **FR-020**: ブーストボタンに絵文字（🚀等）を含めなければならない
- **FR-021**: ブーストボタンにグラデーション背景を適用しなければならない
- **FR-022**: ブーストボタンのタップ時に押下アニメーション（拡大・光る等）を再生しなければならない
- **FR-023**: ブーストがクールダウン中は使用不可であることを視覚的に示さなければならない（半透明化・グレーアウト等）

### Key Entities

- **CooldownIndicator（クールダウンインジケーター）**: ブーストの再使用可能までの進捗を表示するUI要素。進捗率（0〜100%）・表示状態・完了エフェクトの状態を持つ
- **TutorialScreen（チュートリアル画面）**: 操作説明を行うオーバーレイ画面。アニメーションステップ・現在のステップ番号・再生状態を持つ
- **HomeButton（ホームボタン）**: タイトル画面へ遷移するナビゲーション要素。HUDの一部として配置される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: iPad Safariで「あそぶ」ボタンタップ後、100%の確率でBGMが再生開始される
- **SC-002**: 全8ステージでHUD上部にステージ名が視認可能な状態で表示される
- **SC-003**: ステージプレイ中のどの時点でもホームボタン1タップでタイトル画面に3秒以内に遷移できる
- **SC-004**: チュートリアル画面で3つの操作方法（移動・ブースト・目的）がすべてアニメーションで説明される
- **SC-005**: ブーストのクールダウン進捗が視覚的にリアルタイムで確認でき、完了時に光るエフェクトが発生する
- **SC-006**: ブーストボタンのデザインが絵文字・グラデーション・押下アニメーションを含む新デザインに変更される
- **SC-007**: 全UIがひらがな表記で、子供（4〜8歳）が理解可能なデザインである
- **SC-008**: 新機能のUI要素追加によりiPad上でフレームレートが目視で低下しない

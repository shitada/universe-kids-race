# Feature Specification: コンパニオンタイミング修正・シールド楕円化・ブースト炎改善

**Feature Branch**: `008-companion-shield-boost-fix`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "コンパニオンタイミング修正・シールド楕円化・ブースト炎改善"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 🛸 コンパニオンのタイミング修正と仲間演出 (Priority: P1)

現在コンパニオンはステージ開始時に unlockedPlanets に基づいてすべて一括表示されるが、ユーザーは「ステージクリアした瞬間に新しい仲間が加わる」演出を求めている。ステージクリア時に、そのステージの惑星のコンパニオンが初めて登場する専用演出を追加する。クリアメッセージに続いて「{emoji} {name}が なかまに なったよ！」テキストが表示され、新しいコンパニオンがスピン＋拡大アニメーションで登場する。次のステージ以降では、以前クリアした惑星のコンパニオンが従来通りオービットで表示される。

**Why this priority**: ステージクリアの達成感と仲間獲得の喜びを直結させる中核的なユーザー体験改善。現状では仲間が「いつの間にかいる」状態になっており、報酬感が薄い。クリア直後の演出は子どもの達成意欲を大きく刺激し、周回プレイの最大の動機付けとなる。

**Independent Test**: ステージ1をクリアして、クリアメッセージの後に月のコンパニオンが「🌙 つきが なかまに なったよ！」テキストとともにスピン＋拡大アニメーションで登場すること、その後ステージ2を開始したとき月のコンパニオンがオービットで表示されていることを確認する。

**Acceptance Scenarios**:

1. **Given** プレイヤーがステージ1（月）を初めてクリアする, **When** ステージクリア演出が再生される, **Then** 「やったね！」メッセージに続いて「🌙 つきが なかまに なったよ！」テキストが表示される
2. **Given** ステージクリア演出中に仲間テキストが表示された, **When** コンパニオン登場アニメーションが再生される, **Then** 新しいコンパニオンがスピン（回転）しながら拡大して画面に登場する
3. **Given** ステージ1で月のコンパニオンを獲得済み, **When** ステージ2を開始する, **Then** 月のコンパニオンが宇宙船の周りでオービット表示されている（既存動作維持）
4. **Given** ステージ1をすでにクリア済み, **When** ステージ1を再度クリアする, **Then** 仲間獲得演出は表示されず、通常のクリア演出のみが再生される
5. **Given** ステージ5まで全てクリア済み, **When** ステージ6を開始する, **Then** 5体のコンパニオンがオービットで表示され、ステージ6クリア時に6体目の登場演出が再生される

### User Story 2 - 🛡️ エアシールドをブースト専用の楕円形に変更 (Priority: P1)

現在エアシールドは常時表示（通常時は薄い脈動、ブースト時に強調表示）だが、ユーザーは通常時のシールドは不要で、ブースト中のみ楕円形で表示してスピード感を演出したいと考えている。通常時はシールドを非表示にし、ブースト開始時のみ楕円形のシールドを表示する。楕円形は進行方向（Z方向）を伸ばした形状にしてスピード感を出す。

**Why this priority**: ブースト時のビジュアルフィードバックを改善し、速度感を視覚的に強調する重要な演出改善。常時表示のシールドは視覚的なノイズになっており、ブースト専用にすることで通常時の画面をすっきりさせつつ、ブースト時のインパクトを増大させる。

**Independent Test**: 通常走行時にシールドが見えないことを確認し、ブーストを発動してシールドが楕円形で表示されること、ブースト終了後にシールドが非表示に戻ることを確認する。

**Acceptance Scenarios**:

1. **Given** プレイヤーが通常速度で飛行中, **When** 画面を確認する, **Then** エアシールドは表示されていない（非表示状態）
2. **Given** プレイヤーが通常速度で飛行中, **When** ブーストを発動する, **Then** エアシールドが楕円形で表示される（進行方向に伸びた形状）
3. **Given** ブースト中でエアシールドが楕円形に表示されている, **When** ブーストが終了する, **Then** エアシールドが非表示に戻る
4. **Given** ブースト中にエアシールドが表示されている, **When** シールドの形状を確認する, **Then** 横幅は通常サイズ、縦方向はやや扁平、進行方向（前後）に約2倍伸びた楕円形になっている
5. **Given** ブースト中にエアシールドが表示されている, **When** シールドの色と透明度を確認する, **Then** 既存のブースト時と同じ色・不透明度（水色系、半透明）で表示されている

### User Story 3 - 🔥 ブースト炎エフェクトの持続保証・改善 (Priority: P1)

ブースト中の炎パーティクルが途中で途切れる可能性がある。ブースト中は炎パーティクルが毎フレーム確実に放出され、途切れなく持続することを保証する。必要に応じてパーティクルの寿命やサイズを調整し、より豪華な噴射感を演出する。

**Why this priority**: ブーストは子どもにとって最も興奮する瞬間であり、炎エフェクトが途切れると興ざめになる。視覚的な連続性はゲームの没入感に直結し、ブーストの爽快感を維持するために不可欠な修正。

**Independent Test**: ブーストを発動してから終了まで、炎パーティクルが途切れることなく連続的に放出され続けることを目視確認する。

**Acceptance Scenarios**:

1. **Given** プレイヤーがブーストを発動する, **When** ブースト持続時間中に炎エフェクトを観察する, **Then** 炎パーティクルが途切れることなく毎フレーム連続的に放出される
2. **Given** ブースト中に炎パーティクルが放出されている, **When** ブースト開始から終了まで全体を観察する, **Then** パーティクルの密度が一定に保たれ、途中で薄くなったり止まったりしない
3. **Given** ブーストが終了する, **When** 炎エフェクトの終了を確認する, **Then** 放出が停止し、残存パーティクルが自然に消滅する

---

### Edge Cases

- ステージクリア演出中（コンパニオン登場アニメーション再生中）にプレイヤーがタップした場合、アニメーションが中断されず最後まで再生される
- コンパニオン登場演出中にブラウザがバックグラウンドに移行した場合、復帰時にアニメーションが完了状態になっている
- ブースト発動直後にステージクリアした場合、シールドの非表示処理とクリア演出が競合しない
- ブーストが極短時間で終了した場合（ほぼ一瞬）でもシールド表示→非表示の遷移が正しく動作する
- コンパニオン登場演出とブースト炎エフェクトが同時に発生した場合、両方の演出が正しく表示される
- ステージクリア時にコンパニオン獲得状態がセーブされた直後にブラウザがクラッシュしても、データが失われない
- 11体全てのコンパニオンを所持した状態でステージクリアしても、エラーが発生しない（登場演出はスキップ）
- ブースト炎パーティクルとエアシールドが同時に表示される際、視覚的に適切に重なる（炎がシールド内側から出る）

## Requirements *(mandatory)*

### Functional Requirements

#### コンパニオンのタイミング修正と仲間演出

- **FR-001**: ステージクリア時に、そのステージの惑星のコンパニオンが初登場する場合、専用の仲間獲得演出を表示する
- **FR-002**: 仲間獲得演出として「{emoji} {name}が なかまに なったよ！」テキストをクリアメッセージの後に表示する
- **FR-003**: 新しいコンパニオンがスピン（回転）しながら拡大するアニメーションで登場する
- **FR-004**: ゲーム中にコンパニオンを動的に追加できる仕組みを用意し、ステージ番号を指定してコンパニオンを追加できるようにする
- **FR-005**: すでに獲得済みのコンパニオンのステージを再クリアした場合、仲間獲得演出は表示しない
- **FR-006**: 次のステージ以降では、以前クリアした惑星のコンパニオンが従来通りオービット表示される（既存動作維持）

#### エアシールドをブースト専用の楕円形に変更

- **FR-007**: 通常飛行時はエアシールドを非表示にする
- **FR-008**: ブースト開始時にエアシールドを表示し、ブースト終了時に非表示にする
- **FR-009**: エアシールドの形状を楕円形にする（横幅は通常、縦はやや扁平、進行方向に約2倍伸ばす）
- **FR-010**: ブースト時のシールドの色・不透明度は既存と同じ（水色系、半透明）を維持する

#### ブースト炎エフェクトの持続保証・改善

- **FR-011**: ブースト中は炎パーティクル放出処理が毎フレーム確実に呼ばれ、途切れなく放出され続けることを保証する
- **FR-012**: パーティクルの寿命やサイズを必要に応じて調整し、より豪華な噴射感を演出する

### Key Entities

- **CompanionManager**: コンパニオンの管理を担うシステム。動的追加メソッドを持ち、ステージクリア時にコンパニオンを個別追加できる
- **AirShield**: 宇宙船を覆う防護シールドの視覚表現。ブースト時のみ楕円形で表示される
- **BoostFlameEffect**: ブースト中に宇宙船後方から放出される炎パーティクル。リングバッファで管理され、毎フレーム放出される
- **StageScene**: ステージのゲームプレイを管理するシーン。クリア時にコンパニオン獲得演出をトリガーする

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ステージクリア時のコンパニオン登場演出が100%再生され、テキスト表示とアニメーションが途切れない
- **SC-002**: 通常飛行中にエアシールドが画面に表示されない（視覚的にクリーンな状態）
- **SC-003**: ブースト発動から0.5秒以内にエアシールドが楕円形で表示され、ブースト終了後0.5秒以内に非表示になる
- **SC-004**: ブースト持続時間中、炎パーティクルが途切れることなく連続的に放出される（フレーム落ちなし）
- **SC-005**: iPad Safariで全演出（コンパニオン登場・楕円シールド・炎パーティクル）が60fps前後を維持する
- **SC-006**: 子どもがステージクリア時に「仲間が増えた」ことを即座に認識できる（テキスト+アニメーション演出）

## Assumptions

- コンパニオンのデータ構造（惑星テーマカラー・絵文字・名前）は既存の PlanetEncyclopedia 設定を流用する
- ブースト発動・終了のイベントは既存のゲームループから取得できる
- パーティクルの寿命・サイズ調整は既存のリングバッファ（100パーティクル、寿命0.5秒、毎フレーム5個放出）の範囲内で行う
- コンパニオン登場演出の長さは2〜3秒程度を想定（ゲームのテンポを崩さない範囲）
- エアシールドの楕円化は既存の SphereGeometry を scale で変形して実現する（新しいジオメトリは不要）
- 既存のセーブデータ構造（unlockedPlanets）はそのまま活用し、データフォーマットの変更は不要

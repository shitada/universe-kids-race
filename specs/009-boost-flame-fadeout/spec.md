# Feature Specification: ブースト炎の持続保証＆フェードアウト演出

**Feature Branch**: `009-boost-flame-fadeout`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "ブースト炎の持続保証＆フェードアウト演出"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ブースト中の炎が途切れず表示される (Priority: P1)

プレイヤーがブーストを発動すると、宇宙船の後方に炎エフェクトが表示される。現状ではブースト中に炎が途切れることがあり、視覚的に不安定で没入感が損なわれている。ブーストが有効な間は炎が途切れることなく安定して表示され続ける必要がある。

**Why this priority**: ブースト中の炎の途切れはプレイヤーに「ブーストが動作していないのでは？」と誤解させるバグ的挙動であり、最優先で修正すべき。ゲームの基本的なフィードバック品質に直結する。

**Independent Test**: ブーストを発動し、3秒間の全持続時間にわたって炎エフェクトが途切れずに表示されることを目視確認できる。

**Acceptance Scenarios**:

1. **Given** プレイヤーがゲームプレイ中である, **When** ブーストを発動する, **Then** 炎エフェクトがブースト開始と同時に表示され、ブースト有効期間中は途切れることなく連続表示される
2. **Given** ブーストが有効な状態, **When** ブースト持続時間の途中でフレームレートが低下する, **Then** 炎エフェクトは途切れることなく表示され続ける
3. **Given** ブーストが有効な状態, **When** ブースト持続時間の中盤（例: 1.5秒付近）を確認する, **Then** 炎の表示が途切れたり点滅したりしていない

---

### User Story 2 - ブースト終了時に炎が自然にフェードアウトする (Priority: P2)

ブーストが終了する際、炎エフェクトが唐突に消えるのではなく、徐々に弱まって自然に消える演出が必要。ブースト残り時間が少なくなると炎が小さく・少なくなり、終了時にはスムーズに消滅する。これによりプレイヤーにブースト終了が近いことを視覚的に予告でき、ゲーム体験の質が向上する。

**Why this priority**: フェードアウト演出は視覚的な品質向上であり、プレイヤーへのブースト終了予告としても機能する。US1（途切れバグ修正）の次に対応すべき改善項目。

**Independent Test**: ブーストを発動し、終了間際の約0.5秒間で炎が段階的に弱まり、唐突に消えないことを目視確認できる。

**Acceptance Scenarios**:

1. **Given** ブーストが有効で残り時間が0.5秒以下になった, **When** 残り時間がさらに減少する, **Then** 炎エフェクトの放出量が段階的に減少する（全量→75%→50%→25%→0%）
2. **Given** ブーストが有効で残り時間が0.5秒以下になった, **When** 残り時間がさらに減少する, **Then** 炎エフェクトのサイズが段階的に小さくなる
3. **Given** ブーストが有効で残り時間が0.5秒を超えている, **When** プレイヤーが炎を観察する, **Then** 炎は通常の大きさ・量で表示されている（フェードアウトはまだ始まっていない）
4. **Given** ブーストの残り時間が0になった, **When** ブーストが終了する, **Then** 残存する炎パーティクルが自然に消滅し、炎が突然消える現象は発生しない

---

### Edge Cases

- ブースト発動直後（0.5秒以内）にブーストが何らかの理由でキャンセルされた場合、フェードアウトなしで炎が即座に停止しても違和感は少ないため許容する
- 連続してブーストを発動した場合（前回の残存パーティクルがある状態で再発動）、新しいブーストの炎が正常に表示される
- フレームレートが極端に低い状況でも、フェードアウトの段階的な減少が視覚的に認識できる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ブーストが有効な間、炎エフェクトは毎フレーム確実に放出され続けなければならない（途切れの排除）
- **FR-002**: ブースト残り時間の進行割合を取得できる仕組みを提供しなければならない
- **FR-003**: ブースト残り時間が最後の0.5秒に入ったとき、炎パーティクルの放出数が段階的に減少しなければならない
- **FR-004**: ブースト残り時間が最後の0.5秒に入ったとき、炎パーティクルのサイズが段階的に縮小しなければならない
- **FR-005**: フェードアウト中の炎の減少は滑らかで、段階間のジャンプが目立たないものでなければならない
- **FR-006**: フェードアウト開始前（残り0.5秒超）の炎の表示は現行の通常表示と同一でなければならない
- **FR-007**: 既存のパーティクル上限やバッファ方式の動作を損なってはならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ブースト全持続時間（3秒間）にわたり、炎エフェクトの途切れが0回であること
- **SC-002**: ブースト終了時の炎の消滅が0.5秒以上かけて段階的に行われ、即座の消滅が発生しないこと
- **SC-003**: フェードアウト中に炎の放出量が少なくとも3段階以上で減少すること（段階的変化が視認できる）
- **SC-004**: フェードアウト中に炎のサイズが視認できるレベルで縮小すること
- **SC-005**: iPad上で60fpsの描画パフォーマンスが維持されること（既存性能を劣化させない）
- **SC-006**: 連続ブースト発動時にも炎の表示が正常に機能すること

## Assumptions

- ブースト持続時間は現行の3秒を前提とする（将来変更された場合も「最後の0.5秒」のフェードアウト仕様は持続時間に対する絶対値として適用）
- フェードアウトの0.5秒は固定値とし、ブースト持続時間の割合ではなく絶対時間とする
- 炎パーティクルの最大数上限やリングバッファ方式は現行のまま維持する
- ブースト発動後0.5秒未満でのキャンセル時はフェードアウト演出を省略してよい

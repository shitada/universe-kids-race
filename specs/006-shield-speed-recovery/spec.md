# Feature Specification: エアシールドエフェクト追加・隕石衝突後の速度緩やか回復

**Feature Branch**: `006-shield-speed-recovery`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "空気の幕エフェクト追加・隕石衝突後の速度緩やか回復。ブースト中の炎エフェクトに加え常時表示のエアシールドエフェクト追加。隕石衝突時の減速からの回復を段階的にスムーズにする。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 宇宙船が常時エアシールドに包まれて見える (Priority: P1)

プレイヤーがステージをプレイしている間、宇宙船の周りに空気の幕（エアシールド）のようなエフェクトが常時表示される。通常飛行時にもエアシールドが表示されることで、宇宙船がスピード感のある大気のバリアに包まれているように見える。ブースト発動中はエアシールドと既存の炎ジェットパーティクルの両方が同時に表示され、より迫力のあるビジュアルになる。

**Why this priority**: エアシールドは通常飛行時にも常時表示される新しい視覚演出であり、ゲーム全体の見た目の印象を大きく向上させる。ブースト中のみ表示される炎エフェクトとは異なり、プレイ全体の体験に影響する中核的な改善。

**Independent Test**: ステージを開始し、宇宙船の周りにエアシールドエフェクトが表示されていることを確認する。ブーストを発動し、エアシールドと炎パーティクルの両方が同時に表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージが開始される, **When** 宇宙船が通常速度で飛行している, **Then** 宇宙船の周囲にエアシールドエフェクトが表示されている
2. **Given** エアシールドが表示されている状態, **When** ブーストを発動する, **Then** エアシールドと炎ジェットパーティクルの両方が同時に表示される
3. **Given** ブーストが発動中, **When** ブーストが終了する, **Then** 炎ジェットパーティクルは消滅するが、エアシールドは引き続き表示されたままである
4. **Given** 宇宙船が飛行中, **When** 宇宙船が左右に移動する, **Then** エアシールドエフェクトが宇宙船の位置に追従する
5. **Given** エアシールドが表示されている状態, **When** ブースト中にエアシールドを観察する, **Then** エアシールドの見た目がブースト時に強調される（より明るく・大きくなる）

---

### User Story 2 - 隕石衝突後に速度が緩やかに回復する (Priority: P1)

プレイヤーが隕石に衝突すると宇宙船の速度が40%に低下する。3秒間の減速期間が終了した後、速度が瞬時に100%に戻るのではなく、40%から100%へ滑らかに段階的に回復する。これにより唐突な速度変化がなくなり、自然でスムーズなプレイ感覚が得られる。

**Why this priority**: 現在の瞬時回復は体験として不自然であり、プレイヤーにとって唐突に感じる。速度の緩やかな回復はゲームプレイの快適性に直接影響する改善であり、エアシールドと同等に重要。

**Independent Test**: 隕石に衝突し、3秒間の減速後に速度が瞬時ではなく徐々に回復することを体感で確認する。

**Acceptance Scenarios**:

1. **Given** 隕石に衝突し速度が40%に低下している, **When** 3秒間の減速期間が終了する, **Then** 速度が40%から100%へ瞬時ではなく滑らかに徐々に回復する
2. **Given** 速度回復中（40%→100%の遷移途中）, **When** 回復の進行を観察する, **Then** 速度が徐々に上がり、途中で急な変化がない
3. **Given** 速度回復中, **When** 回復が完了する, **Then** 速度が正確に100%（通常速度）に到達し、それ以上上がらない
4. **Given** 速度回復中, **When** 再び隕石に衝突する, **Then** 速度が再度40%に低下し、新たな3秒間の減速期間が開始される
5. **Given** 速度回復中, **When** ブーストを発動する, **Then** ブーストの速度倍率が回復途中の速度に対して正しく適用される

---

### Edge Cases

- エアシールドが表示中に隕石に衝突し減速状態になった場合、エアシールドは引き続き表示される
- 速度回復中にステージクリアした場合、速度回復状態がリセットされる
- 速度回復中にブーストを発動した場合の速度計算が正しく行われる（回復途中の速度 × ブースト倍率）
- ブースト中に隕石に衝突した場合、ブーストが終了し減速状態になり、エアシールドは引き続き表示される
- 連続して複数回隕石に衝突した場合（回復中に再衝突）、減速タイマーがリセットされ40%から再度カウントダウンが始まる
- エアシールドエフェクトがiPad Safariで60fpsを維持できるパフォーマンスであること
- 減速状態から回復中にゲームが一時停止・再開された場合、回復が正しく継続する
- エアシールドとブースト炎パーティクルが同時に表示されてもパーティクル数が過剰にならないこと

## Requirements *(mandatory)*

### Functional Requirements

#### エアシールドエフェクト

- **FR-001**: 宇宙船の周囲にエアシールド（空気の幕）エフェクトが通常飛行時から常時表示される
- **FR-002**: エアシールドは半透明の発光するバリアのような見た目で、宇宙船を包み込むように表示される
- **FR-003**: エアシールドは宇宙船の移動に追従する
- **FR-004**: ブースト発動中はエアシールドと既存の炎ジェットパーティクルの両方が同時に表示される
- **FR-005**: ブースト中はエアシールドの視覚的な強調が行われる（より明るく、またはより大きく表示される）
- **FR-006**: ブースト終了時、炎ジェットパーティクルは消滅するが、エアシールドは消えずに通常表示に戻る
- **FR-007**: 隕石衝突による減速状態でもエアシールドは表示され続ける

#### 速度の緩やか回復

- **FR-008**: 隕石衝突による3秒間の減速期間終了後、速度が40%から100%へ瞬時ではなく段階的に回復する
- **FR-009**: 速度回復は滑らかな曲線（イージング）で行われ、急激な変化が生じない
- **FR-010**: 速度回復の遷移時間は約1秒間とする
- **FR-011**: 速度回復が完了すると正確に通常速度（100%）に到達する
- **FR-012**: 速度回復中に再び隕石に衝突した場合、速度が40%にリセットされ新たな3秒間の減速期間が開始される
- **FR-013**: 速度回復中にブーストを発動した場合、現在の回復途中の速度にブースト倍率が正しく適用される

### Key Entities

- **エアシールド（Air Shield）**: 宇宙船を包み込む常時表示の視覚演出。通常時とブースト時で見た目の強度が変化する。宇宙船の位置に追従する
- **速度回復状態（Speed Recovery State）**: 減速期間終了後から通常速度に戻るまでの遷移状態。回復開始速度（40%）、目標速度（100%）、遷移時間（約1秒）、経過時間を持つ

## Assumptions

- エアシールドの色は青白い半透明（宇宙空間における大気圏のイメージ）を想定
- ブースト時のエアシールド強調は、通常時の1.5〜2倍程度の明るさ・サイズを想定
- 速度回復のイージングはイーズアウト（最初は速く、終盤はゆっくり）を想定し、自然な加速感を演出する
- 速度回復の遷移時間（約1秒）は実際のプレイテストで調整される可能性がある
- iPad Safariでのパフォーマンスを考慮し、エアシールドのパーティクル数は最小限に抑える
- 既存の炎ジェットパーティクル（リングバッファ100パーティクル、AdditiveBlending）には変更を加えない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 通常飛行時に宇宙船の周囲にエアシールドエフェクトが常時表示され、宇宙船のスピード感・存在感が視覚的に向上している
- **SC-002**: ブースト中にエアシールドと炎ジェットの両方が同時に表示され、ブースト体験がよりリッチになっている
- **SC-003**: 隕石衝突後の速度回復が滑らかで、プレイヤーが唐突な速度変化を感じない
- **SC-004**: 速度回復の遷移が約1秒で完了し、ゲームテンポを損なわない
- **SC-005**: iPad Safariで全エフェクト表示中も60fpsが維持される
- **SC-006**: エアシールドの見た目から「速い」「大気のバリア」という印象をプレイヤーが直感的に受ける

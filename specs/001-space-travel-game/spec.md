# Feature Specification: うちゅうの たび — 宇宙船キッズゲーム

**Feature Branch**: `001-space-travel-game`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "宇宙船を操作して星々を旅する子供向けゲーム（対象年齢5〜10歳、iPad専用）。タイトル画面→ステージ（月・火星・土星）→エンディングの流れで、星を集め隕石を避けながらゴールの惑星を目指す。"

## Assumptions

- 対象デバイスはiPad専用（横向き固定）
- 対象年齢5〜10歳のため、文字はひらがな中心で大きく表示する
- ゲームオーバーは存在しない（隕石に当たってもスピードが落ちるだけで、必ずゴールできる）
- ステージは順番にプレイし、ステージ選択画面は設けない（シンプルさ優先）
- スコアやプレイデータのオンライン保存・ランキング機能は含まない
- サウンド・BGMは仕様範囲に含む（子供が楽しめるポップな音）
- 認証やアカウント機能は不要（起動してすぐ遊べる）
- セーブ機能はアプリ終了時にステージ進行状況をローカル保存する（途中のステージからやり直せる）

## User Scenarios & Testing *(mandatory)*

### User Story 1 — タイトル画面からステージ1を遊ぶ (Priority: P1)

子ども（5〜10歳）がアプリを起動すると、宇宙をイメージしたタイトル画面にゲーム名「うちゅうの たび」と大きな「あそぶ」ボタンが表示される。「あそぶ」ボタンをタッチするとステージ1が始まり、宇宙船が画面奥に向かって自動で前進する。画面の左右をタッチすると宇宙船が左右に移動し、星を集めながら月を目指す。月に到着するとクリア演出が表示される。

**Why this priority**: ゲームのコアループ（起動→操作→ゴール）を体験できる最小単位。これが動けばゲームとして成立する。

**Independent Test**: タイトル画面の「あそぶ」ボタンをタッチし、左右操作で宇宙船を動かし、月に到着してクリア演出が表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** アプリを起動した状態, **When** タイトル画面が表示される, **Then** ゲーム名「うちゅうの たび」と「あそぶ」ボタンがひらがなで大きく表示される
2. **Given** タイトル画面が表示されている, **When** 「あそぶ」ボタンをタッチする, **Then** ステージ1（月へ向かう宇宙空間）が始まり宇宙船が自動で前進する
3. **Given** ステージ1がプレイ中, **When** 画面左側をタッチする, **Then** 宇宙船が左に移動する
4. **Given** ステージ1がプレイ中, **When** 画面右側をタッチする, **Then** 宇宙船が右に移動する
5. **Given** 宇宙船がステージ1の終端に到達した, **When** ゴールの月に到着する, **Then** 「やったね！」というクリア演出と集めた星の数が表示される

---

### User Story 2 — 星を集めてスコアを稼ぐ (Priority: P1)

プレイ中に宇宙空間に浮かぶ星に宇宙船が触れるとスコアが加算され、集めた星の数が増える。まれに出現する虹色の星はボーナススコアがもらえる。画面上部にスコアと集めた星の数が常に表示される。

**Why this priority**: スコアと収集はゲームの達成感・モチベーションの根幹であり、コアループに不可欠。

**Independent Test**: ステージをプレイし、通常の星と虹色の星に接触してスコアと星の数が正しく加算されることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** 宇宙船が通常の星に触れる, **Then** スコアが加算され、集めた星の数が1つ増える
2. **Given** ステージをプレイ中, **When** 宇宙船が虹色の星に触れる, **Then** 通常の星より大きいボーナススコアが加算され、集めた星の数が1つ増える
3. **Given** ステージをプレイ中, **When** 画面を確認する, **Then** 現在のスコアと集めた星の数が画面上部に常時表示されている
4. **Given** 虹色の星が出現する, **When** 出現頻度を観察する, **Then** 虹色の星は通常の星よりもまれに出現する

---

### User Story 3 — 隕石を避ける (Priority: P1)

プレイ中に隕石が飛来し、宇宙船に当たるとスピードが一時的に落ちる。ゲームオーバーにはならず、スピードは数秒後に回復する。子どもが悲しまないよう、ダメージ演出は軽くかわいらしいものにする。

**Why this priority**: 障害物はゲームの緊張感と楽しさを生むコア要素。ゲームオーバーなしの設計は対象年齢への配慮として必須。

**Independent Test**: ステージ中に意図的に隕石に当たり、スピードダウン→回復の挙動を確認する。ゲームが終了しないことを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** 宇宙船が隕石に接触する, **Then** 宇宙船の前進スピードが一時的に落ちる
2. **Given** 宇宙船が隕石に当たってスピードが落ちた, **When** 数秒が経過する, **Then** スピードが元に戻る
3. **Given** 宇宙船が隕石に何度当たっても, **When** ゲーム状態を確認する, **Then** ゲームオーバーにはならずプレイを継続できる
4. **Given** 宇宙船が隕石に接触した, **When** 演出を確認する, **Then** 軽くかわいらしいリアクション演出が再生される（恐怖感を与えない）

---

### User Story 4 — ブーストで加速する (Priority: P2)

画面上のブーストボタンをタッチすると、宇宙船が一時的に速いスピードで前進する。ブースト中はエフェクトが表示され、終了するとエフェクトが消えて通常速度に戻る。

**Why this priority**: ゲームプレイに変化と爽快感を加える要素。コアループは成立した上での拡張機能。

**Independent Test**: ブーストボタンを押して加速を確認し、一定時間後に通常速度に戻ることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** ブーストボタンをタッチする, **Then** 宇宙船が一時的に速いスピードで前進する
2. **Given** ブースト中, **When** ブーストの制限時間が経過する, **Then** 宇宙船が通常速度に戻りエフェクトが消える
3. **Given** ブースト中, **When** 画面を確認する, **Then** 加速中であることを示すエフェクトが表示されている

---

### User Story 5 — 3ステージを順にクリアしてエンディングを見る (Priority: P2)

ステージ1（月）をクリアするとステージ2（火星）が始まり、ステージ2をクリアするとステージ3（土星）が始まる。各ステージは難易度が段階的に上がる（隕石の量が増える）。ステージ3をクリアすると「うちゅうの たびは おしまい！」というエンディング画面が表示され、総スコアが表示される。

**Why this priority**: ゲーム全体の流れと達成感の完結に必要。P1でコアループが成立した後に構築する。

**Independent Test**: ステージ1からステージ3まで順番にクリアし、各ステージ間の遷移とエンディング画面の表示を確認する。

**Acceptance Scenarios**:

1. **Given** ステージ1をクリアした, **When** クリア演出が終わる, **Then** ステージ2（火星）が始まる
2. **Given** ステージ2をクリアした, **When** クリア演出が終わる, **Then** ステージ3（土星）が始まる
3. **Given** ステージ3をクリアした, **When** クリア演出が終わる, **Then** 「うちゅうの たびは おしまい！」というエンディング画面と総スコアが表示される
4. **Given** ステージ1をプレイ中, **When** 隕石の出現頻度を観察する, **Then** 隕石は少なく簡単である
5. **Given** ステージ2をプレイ中, **When** 隕石の出現頻度を観察する, **Then** ステージ1より隕石が多く中程度の難易度である
6. **Given** ステージ3をプレイ中, **When** 隕石の出現頻度を観察する, **Then** 隕石が多く難しい

---

### User Story 6 — サウンドとBGMで没入感を得る (Priority: P3)

ゲーム中にポップなBGMが流れ、星を集めたとき・隕石に当たったとき・クリア時にそれぞれ効果音が鳴る。子どもが楽しめる明るいトーンの音を使用する。

**Why this priority**: 音はゲーム体験を豊かにするが、音なしでもゲームとして成立する。

**Independent Test**: 各場面（BGM、星取得、隕石接触、クリア）で適切な音が再生されることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージをプレイ中, **When** BGMを確認する, **Then** 宇宙をイメージしたポップなBGMが流れている
2. **Given** ステージをプレイ中, **When** 宇宙船が星に触れる, **Then** 明るい効果音が鳴る
3. **Given** ステージをプレイ中, **When** 宇宙船が隕石に当たる, **Then** 軽いリアクション効果音が鳴る（恐怖感のない音）
4. **Given** ステージをクリアした, **When** クリア演出が表示される, **Then** お祝い感のある効果音が鳴る

---

### User Story 7 — 進行状況が保存される (Priority: P3)

アプリを途中で閉じても、次回起動時にクリア済みのステージの次から再開できる。エンディングを見た後は、タイトル画面からもう一度最初からプレイできる。

**Why this priority**: 子どもが一度にすべてクリアできなくても体験を維持できる。ただし保存なしでも遊ぶこと自体は可能。

**Independent Test**: ステージ1クリア後にアプリを終了し、再起動してステージ2から始まることを確認する。

**Acceptance Scenarios**:

1. **Given** ステージ1をクリアした状態でアプリを終了した, **When** アプリを再起動する, **Then** ステージ2から始められる
2. **Given** 全ステージをクリアしてエンディングを見た, **When** タイトル画面に戻る, **Then** 「あそぶ」ボタンでステージ1から最初からプレイできる

---

### Edge Cases

- 宇宙船が画面の左端・右端に達した場合：画面外には出ず、端で止まる
- ブースト中に隕石に当たった場合：ブーストが解除されスピードダウン状態になる
- 星と隕石が同時に重なっている場合：両方の効果（スコア加算とスピードダウン）が適用される
- プレイ中にアプリがバックグラウンドに回った場合：ゲームが自動的に一時停止し、復帰時に再開できる
- 画面を左右同時にタッチした場合：宇宙船は動かない（左右が相殺される）

## Requirements *(mandatory)*

### Functional Requirements

**タイトル画面**
- **FR-001**: ゲーム起動時にタイトル画面を表示し、ゲーム名「うちゅうの たび」と「あそぶ」ボタンを表示しなければならない
- **FR-002**: 「あそぶ」ボタンタッチでステージ1（未クリアの場合）または最後にクリアしたステージの次のステージを開始しなければならない

**宇宙船操作**
- **FR-003**: 宇宙船はステージ開始と同時に画面奥方向へ自動で前進しなければならない
- **FR-004**: 画面左側タッチで宇宙船が左へ、画面右側タッチで宇宙船が右へ移動しなければならない
- **FR-005**: 宇宙船は画面の左右端を超えて移動できてはならない
- **FR-006**: ブーストボタンをタッチすると宇宙船が一時的に高速で前進し、制限時間後に通常速度に戻らなければならない

**星の収集**
- **FR-007**: ステージ中に通常の星が配置され、宇宙船が触れるとスコアが加算され集めた星の数が1つ増えなければならない
- **FR-008**: 虹色の星（レア）がまれに出現し、接触時に通常の星より高いボーナススコアを付与しなければならない

**隕石**
- **FR-009**: ステージ中に隕石が飛来し、宇宙船が接触するとスピードが一時的に低下しなければならない
- **FR-010**: 隕石接触によるゲームオーバーは発生してはならない
- **FR-011**: スピード低下は数秒後に自動回復しなければならない

**UI表示**
- **FR-012**: プレイ中は画面上部にスコアと集めた星の数を常時表示しなければならない
- **FR-013**: テキストはすべてひらがな中心で、対象年齢（5〜10歳）が読めるサイズで表示しなければならない

**ステージ進行**
- **FR-014**: ステージ1（月）は隕石が少なく簡単、ステージ2（火星）は中程度、ステージ3（土星）は隕石が多く難しい構成にしなければならない
- **FR-015**: 各ステージのゴール惑星に到着したら「やったね！」と集めた星の数を含むクリア演出を表示しなければならない
- **FR-016**: ステージ3クリア後に「うちゅうの たびは おしまい！」と総スコアを含むエンディング画面を表示しなければならない
- **FR-017**: エンディング後はタイトル画面に戻り、最初からプレイできなければならない

**データ保存**
- **FR-018**: ステージクリア状況をローカルに保存し、アプリ再起動時に続きのステージからプレイできなければならない

**一時停止**
- **FR-019**: アプリがバックグラウンドに移行した際にゲームを自動的に一時停止し、復帰時に再開できなければならない

**サウンド**
- **FR-020**: ステージ中にBGMを再生しなければならない
- **FR-021**: 星取得・隕石接触・クリア時にそれぞれ効果音を再生しなければならない

**対象デバイス**
- **FR-022**: iPad専用の横向き固定レイアウトで表示しなければならない

### Key Entities

- **宇宙船（Spaceship）**: プレイヤーが操作するキャラクター。位置（左右）、速度状態（通常/ブースト中/スピードダウン中）を持つ
- **星（Star）**: 収集対象アイテム。通常星とレアの虹色星の2種類。それぞれスコア値が異なる
- **隕石（Meteorite）**: 障害物。接触するとスピードダウンを引き起こす。ステージごとに出現頻度が変わる
- **ステージ（Stage）**: ゲームの区切り。番号（1〜3）、目的地の惑星（月・火星・土星）、難易度（隕石頻度）を持つ
- **スコア（Score）**: 星の収集で加算される数値。ステージごとのスコアと総スコアを管理する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 5歳の子どもが初めてプレイして、説明なしで3分以内に操作方法を理解しゲームを開始できる
- **SC-002**: 対象年齢の子どもがステージ1を5分以内にクリアできる
- **SC-003**: 全3ステージを通してプレイした場合、15分以内にエンディングに到達できる
- **SC-004**: プレイ中にゲームオーバーが一度も発生しない（挫折ポイントがゼロ）
- **SC-005**: 画面上のすべてのテキストが対象年齢（5〜10歳）にとって読みやすい（ひらがな中心、十分な文字サイズ）
- **SC-006**: アプリ終了後に再起動した際、前回のステージ進行状況が保持されている
- **SC-007**: 子どものテストプレイで、80%以上が「もう一回遊びたい」と回答する
